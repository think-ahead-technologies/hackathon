# pipeline — model supply chain + device provisioning

Board-free Platform tooling: **everything right of the model artifact**. Two concerns live here:

1. **Model supply chain** — gate a Vela-compiled model against the edge contract, package + sign it,
   push it to a sovereign OCI registry, GitOps-promote it, and stream its bytes to devices.
2. **Secure-by-default fabric** — issue a unique per-device NATS credential and mint the TLS chain
   (EU CRA: no shared/default secret, confidentiality + integrity in transit).

All of it is driven by `make` targets in `dashboard/Makefile`; this README is the file-level map. The
*conceptual* write-up (trust domains, two signatures, two delivery planes) is in the
[root README](../README.md#model-pipeline-vela-gate--sign--gitops-promote). The ML that produces the
artifact this consumes lives in [`wear_detector/`](../wear_detector/); the device that receives it lives
in [`firmware/`](../firmware/).

## Model supply chain

```
Vela summary CSV ─► gate ─► manifest + sha256 ─► oras push → zot ─► cosign sign (in-registry)
                                                                          │
  desired-state.json (bump version) ─► promote: observed vs desired → cosign verify → Contract C event
                                                                          │
                          publish_artifact ◄── oras pull ──┘   → chunked Contract C frames on NATS
```

| File | Role | Driven by |
|---|---|---|
| `gate.py` | Deployability gate — fails the build if the Vela summary violates the edge contract (SRAM / inference latency / CPU-fallback ratio / flatbuffer size). | `make gate` (`make gate MODEL=bad` shows rejection) |
| `vela.policy.json` | The gate policy: CSV column mapping + thresholds. The *edge contract* a model must satisfy. | — |
| `samples/` | Committed sample Vela summaries (`vela-summary.csv` passes, `…-bad.csv` fails) so the gate runs without a real `.tflite`. | — |
| `package.py` | Build the device manifest (Contract A) + sha256 from the artifact. The Makefile then `cosign sign-blob`s it. | `make package` |
| `model-meta.json` | **Contract A** input — the model contract ML hands over (emitted by `wear_detector/export`); `package.py` merges in version/sha256/size. | — |
| `der2raw.py` | Convert the cosign `sign-blob` signature (base64 ASN.1 DER) → raw `r‖s` P1363 (64 B) so the device's PSA `psa_verify_hash` can check it. | `make package` |
| `promote.py` | GitOps reconciler — desired (`desired-state.json`) vs observed (`model_events`), `cosign verify`s the registry artifact, publishes the Contract C deploy *event*. Refuses to deploy unless verify passes. | `make promote` |
| `desired-state.json` | GitOps desired state: the model version each line should run. Bump a value → `make promote`. | — |
| `deploy_frame.py` | Chunk + frame a packaged model into **Contract C** frames. Wire format mirrors `firmware/include/deploy.h` byte-for-byte — keep the two in lockstep (`firmware/ make crosslang` is the guard). | imported by `publish_artifact.py` |
| `publish_artifact.py` | Gateway-side bridge: pull the desired model from the registry (`oras pull`) and stream its *bytes* to devices as Contract C frames on `models.<line>.artifact`. | `make deploy-artifact` |
| `zot-config.json` | Config for **zot**, the self-hosted CNCF OCI registry (TLS) that holds signed artifacts. | `make up` |

## Secure-by-default fabric (CRA)

| File | Role | Driven by |
|---|---|---|
| `provision.py` | Issue a unique Ed25519 nkey per device/service, render a least-privilege NATS config (each identity may only touch its own subjects). No shared/default credential. | `make provision` |
| `nkey.py` | NATS nkey (Ed25519) codec — seed + public-key wire format. Real crypto, no service. | (imported) |
| `tls_certs.py` | Mint the NATS TLS chain — self-signed CA + server cert; clients verify the CA. CA private key is never persisted (re-minted each run). | `make provision` |
| `fleet.json` | Fleet inventory — devices + services. Add a board here and re-run `provision`; permissions derive from role + line + id. | `make provision` |

`make verify-auth` boots a throwaway `nats-server` under the provisioned config and proves the four
properties (authorized publish OK; cross-device denied; anonymous denied; untrusted/plaintext rejected
at TLS). `make up-secure` runs the whole stack over TLS + auth.

## Keys, certs, build outputs

`make keygen` (one-time) produces the cosign keypair + the zot TLS cert. Gitignored, never committed:

- `cosign.key` — cosign **private** signing key (`cosign.pub` *is* committed, for verification).
- `certs/` — zot TLS cert/key (throwaway, local demo registry).
- `build/`, `pull/` — packaging outputs and registry pulls (regenerated by `make package` /
  `make deploy-artifact`).

## Tests

```bash
make test-pipeline    # from dashboard/ — the whole suite
```

Pure logic, no infra: the gate policy (`test_gate.py`), the deploy wire format (`test_deploy_frame.py`,
cross-checked against the C parser by `firmware/ make crosslang`), the DER→raw conversion
(`test_der2raw.py`), the nkey codec (`test_nkey.py`), provisioning + permission derivation
(`test_provision.py`), and the TLS chain (`test_tls_certs.py`).

## Python deps

`requirements.txt` — `cryptography` (backs the Ed25519 nkey codec and the TLS chain). External tooling
(`oras`, `cosign`) is pinned in containers by the Makefile, not installed here.
